Почему HTML считает, что «BruceWillis» — это цвет?

Атрибут color тега может принимать значения в форматах:

  • RGB, например, "rgb(15, 92, 50)"
  • HTML hex, например, "#0F5C32"
  • colorname, например, "black"

Если в атрибут тега передано некорректное значение, браузер не может понять что это за значение и пытается преобразовать его в шестнадцатеричный код с помощью алгоритма, где пропускаемые цифры распознаются как 0, некорректные символы распознаются как 0.

Исходя из вышеперечисленной информации, получается следующее:

  1. Заменяем все неправильные шестнадцатеричные символы нулями 0, BruceWillis превращается в B00ce000000
  2. “Добиваем” число нулями до количества знаков, делимого на 3 (11 -> 12), получаем B00ce0000000
  3. Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: (B00c, e000, 0000)
  4. Обрезаем каждый из аргументов справа, оставляя 2 символа, и получаем результат: #B0e000 или он же rgb(176, 224, 0).

Аналогично с #CANDY-> #CA0D00, rgb(202, 13, 0)

С другими интересными словами тоже сработает
